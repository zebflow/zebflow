# SQLite in Zebflow

SQLite is the lightweight relational option inside Zebflow projects.

Use it when you want:

- a simple structured relational store
- small to medium local application data
- SQL without running a separate database server

## Good fit

- settings tables
- small admin tools
- local caches
- relational prototypes

## When not to pick it first

If your project needs:

- graph traversal
- vector search
- spatial querying
- multimodel relations

then Sekejap is usually the stronger project-default choice.

SQLite stays valuable because it is small, predictable, and familiar.

# Project Management

After you understand the app model, the next layer is operating the project itself.

## Main areas

- Settings
- credentials
- database connections
- files
- git
- session / MCP access
- interface surfaces in Studio

## What matters in practice

- your project repo is the working source of truth
- git tracks the project workspace under `repo/`
- settings and credentials shape how the project can run
- Studio gives a unified surface for building, reviewing, and operating the project

For deeper operational guidance, see:

- `help("platform/operations")`
- `help("platform/workflow")`
